# 🔐 GitHub Secrets Setup Guide
# ════════════════════════════════════════════════════════════════
# Go to: GitHub → your repo → Settings → Secrets and variables → Actions → New repository secret
# Add each secret below:
# ════════════════════════════════════════════════════════════════

# ── Upstox credentials ────────────────────────────────────────
UPSTOX_CLIENT_ID        → Your Upstox app API key
                          Get from: upstox.com/developer/apps → your app → API key

UPSTOX_CLIENT_SECRET    → Your Upstox app API secret
                          Get from: upstox.com/developer/apps → your app → API secret

UPSTOX_REDIRECT_URI     → The redirect URI set in your Upstox app
                          Example: https://pocketrs-pro.vercel.app/callback
                          (Must exactly match what's in your Upstox app settings)

UPSTOX_MOBILE           → Your Upstox account mobile number (10 digits, no +91)
                          Example: 9876543210

UPSTOX_PIN              → Your Upstox 6-digit login PIN

UPSTOX_TOTP_SECRET      → Your TOTP base32 secret key
                          Get this when you set up 2FA in Upstox:
                          Upstox App → Profile → Security → Enable TOTP
                          Look for "Secret key" or "Manual entry key" — it's base32 like:
                          JBSWY3DPEHPK3PXP
                          ⚠️ Save this when you first enable TOTP — you can't retrieve it later

# ── Vercel credentials ────────────────────────────────────────
VERCEL_TOKEN            → Vercel personal access token
                          Get from: vercel.com/account/tokens → Create token
                          Name it "pocketrs-github-actions"

VERCEL_PROJECT_ID       → Your Vercel project ID
                          Get from: vercel.com → your project → Settings → General
                          Look for "Project ID" at the bottom

VERCEL_TEAM_ID          → Your Vercel team ID (leave blank if personal account)
                          Get from: vercel.com → Settings → General → Team ID

# ── Supabase credentials ──────────────────────────────────────
SUPABASE_URL            → Your Supabase project URL
                          Get from: supabase.com → your project → Settings → API → Project URL
                          Example: https://abcdefgh.supabase.co

SUPABASE_SERVICE_KEY    → Supabase SERVICE ROLE key (NOT the anon key)
                          ⚠️ This key bypasses RLS — keep it secret, never expose in frontend
                          Get from: supabase.com → your project → Settings → API → service_role key

# ════════════════════════════════════════════════════════════════
# How the auto-refresh works:
# ════════════════════════════════════════════════════════════════
# 1. GitHub Actions runs every weekday at 8:30 AM IST (3:00 AM UTC)
# 2. Script opens headless Chromium browser
# 3. Navigates to Upstox login page
# 4. Enters your mobile, PIN, and auto-generates TOTP code
# 5. Captures the OAuth authorization code from the redirect
# 6. Exchanges code for fresh access token via Upstox API
# 7. Saves token to Supabase owner_token table
# 8. Updates Vercel VITE_OWNER_UPSTOX_TOKEN env var
# 9. App reads fresh token from Supabase on next page load — no redeploy needed!

# ════════════════════════════════════════════════════════════════
# Manual trigger:
# ════════════════════════════════════════════════════════════════
# GitHub → your repo → Actions → "Refresh Upstox Token" → Run workflow
# Use this if the cron fails or you need a token immediately.

# ════════════════════════════════════════════════════════════════
# Monitoring:
# ════════════════════════════════════════════════════════════════
# GitHub → your repo → Actions → "Refresh Upstox Token"
# Green checkmark = success, Red X = failed (check logs)
# You'll also get an email notification from GitHub if the workflow fails.
